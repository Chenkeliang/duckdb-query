Why DuckQuery + DuckDB

将 DuckDB 用作本地分析引擎，配合 DuckQuery 的无代码操作与 SQL 编辑器， 打造面向业务团队与数据工程团队的统一数据工具。

多源数据 · 秒变 DuckDB 表

Excel / CSV / JSONL / Parquet / 剪贴板 / 远程文件全部支持， 自动映射为 DuckDB 表结构并保存列统计信息，随时 JOIN。

DuckDB SQL & 可视化双模式

支持窗口函数、PIVOT、JSON 等 DuckDB 高级语法；图形化面板内置类型检查、 TRY_CAST 建议，自动生成可靠 SQL。

本地部署 · 数据零外泄

Docker / 源码快速部署，DuckDB 在本地执行查询。敏感数据无需上传云端， 满足企业安全要求。

核心能力

覆盖数据接入、治理、分析与导出全流程，充分释放 DuckDB 的潜力。

跨源关联分析

将 MySQL / PostgreSQL / 文件数据一次性加载到 DuckDB， 在图形化界面配置 JOIN、聚合、透视分析，并可随时切换到 SQL 编辑模式。

智能类型冲突处理

DuckQuery 自动检测 DuckDB 中的类型冲突并给出 TRY_CAST 建议， 解决多源数据类型不一致带来的报错。

异步任务 & 导出

大查询可后台执行，提供进度状态、日志与结果下载。支持将查询结果一键保存为 DuckDB 表或导出为 Parquet/CSV。

现代化 UI 体验

统一主题、智能提示、错误翻译、直观的 Pivot 配置，让分析师在 DuckDB 上的操作与使用 Excel 一样顺滑。

5 分钟上手 DuckDB 分析

快速开始

  1. 克隆仓库并运行 ./quick-start.sh，自动拉起 DuckDB 后端 + 前端。
  2. 上传 Excel/CSV 或连接外部数据库，DuckQuery 自动在 DuckDB 中建表。
  3. 在图形化界面拖拽字段或切换到 SQL 编辑器执行 DuckDB 查询。
  4. 将结果保存为 DuckDB 表或导出为 Parquet/CSV，分享图表或下载。

想了解更多？阅读 DuckDB 快速上手指南 和 DuckDB 集成手册。

临时数仓 & 报表
